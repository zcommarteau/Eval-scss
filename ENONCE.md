# 📝 Projet : Créer un tutoriel SCSS

## Contexte

Tu viens de rejoindre l’entreprise comme référent CSS.
Les développeurs juniors maîtrisent le CSS basique, mais pas encore :
- le CSS avancé
- la méthodologie BEM
- le préprocesseur SCSS

La direction te demande de créer une documentation interne simple, claire et structurée pour leur permettre d’apprendre rapidement et d’adopter de bonnes pratiques communes.

**Ta mission : créer un support de cours complet sous forme de fichier(s) README.**

---

## Ce qu'on attend de toi

Un vrai document pédagogique, rédigé **avec tes propres mots** de la manière la plus claire et compréhensible possible. Exactement comme si tu l'écrivais pour quelqu'un qui n'y connaît rien en css avancé.  
Pas de définitions récitées par cœur. Pas de copié-collé.

---

## Sujets obligatoires à couvrir

Ton tutoriel doit aborder **tous** ces sujets, dans l'ordre et de la façon qui te semble le plus logique pour un débutant :

- Informations *(C'est quoi SCSS ? Pourquoi l'utiliser ?)*
- Méthode BEM
- Préprocesseur, c'est quoi ?
- Installation : NPM & package manager
- Syntaxe SCSS
- Les variables
- Mixins
- Fonctions
- Collections — List & Map
- Boucles — `@for`, `@each`, `@while`
- Multi-fichiers — `@use` & `@forward`
- Architecture 7-1

---

## Règles importantes

### ✅ Sources autorisées
- Tes anciens projets CSS / SCSS personnels
- Le fichier `EXEMPLES.md` fourni *(syntaxes de référence uniquement — pas d'explication dedans)*
- Tes propres tests dans un projet sandbox ou un ancien projet si tu as un doute sur un comportement

### ❌ Sources interdites
- Internet
- IA (ChatGPT, Claude, Gemini, etc.)

---

## Mise en forme

Tout est autorisé pour rendre ton tuto le plus clair possible :

- **Plusieurs fichiers README** si tu veux découper par thème
- **Images** — pour les captures de code, utilise [CodeSnap](https://marketplace.visualstudio.com/items?itemName=adpyke.codesnap) (extension VS Code)
- **Schémas** — pour les schémas visuels, utilise [Excalidraw](https://excalidraw.com)
- Tableaux, listes, titres, gras, italique... tout ce que Markdown permet

> 💡 Si tu n'es pas à l'aise avec la syntaxe Markdown/README, consulte le fichier `README_SYNTAX.md` fourni.

---

## Critères d'évaluation

| Critère | Ce qu'on regarde |
|---|---|
| Couverture des sujets | Tous les thèmes sont traités |
| Compréhension | Les explications sont dans tes mots, pas récitées |
| Pédagogie | Un vrai débutant pourrait suivre ton tuto |
| Exemples | Chaque concept est illustré avec du code concret |
| Clarté | Le document est lisible et bien structuré |
